Uses case name      : Check Availability

Participating actors: User

Entry Condition     : EditReservation or MakeReservation initiates

Exit Condition      : Approve or reject reservation.

Flow of Events      : 1. Reservation form is checked with the HotelDatabase to see if it's valid.
                      2. If valid, a rejection flag is sent to EditReservation or MakeReservation.
                         Else, aconfirmation flag is sent.

Object              : reservation form, hotel database
